# DevPilot Sandbox — 下版本更新计划

> 当前版本: v0.1.0 (核心引擎) / v0.2.1 (VSCode 扩展)

---

## v0.2.0 — 扩展与集成 (4 周)

**目标**: VSCode 扩展稳定可用 + 部署标准化

### 1. VSCode 扩展稳定性
| 任务 | 说明 |
|------|------|
| 扩展激活日志 | 添加 `console.log` 和诊断通道，方便排查激活问题 |
| 自动重连增强 | 当前 30 秒轮询 → 指数退避重试 (1s, 2s, 4s, 8s...) |
| 连接状态通知 | 首次连接成功 / 断开时弹出通知 |
| 角色选择 QuickPick | 输入框 → 下拉选择可用角色 |
| 状态栏图标 | 添加自定义 🛡️ 图标，替代默认图标 |
| 扩展配置面板 | VSCode 设置 UI 中显示 baseUrl / defaultRole 的说明 |
| `.vsix` 自动构建 | 添加到 CI 流程，提交自动打包 |

### 2. 部署标准化
| 任务 | 说明 |
|------|------|
| Docker Compose 验证 | 在有 Docker 的环境中测试构建和启动 |
| `docker-compose.prod.yml` | 生产级部署 (资源限制、日志轮转、健康检查) |
| `.env` 模板 | 完善环境变量：端口、API 密钥、认证开关、JWT 密钥 |
| Windows 服务安装脚本 | 用 `sc.exe` 或 `nssm` 将 Sandbox 注册为 Windows 服务 |

### 3. Web 控制台完善
| 任务 | 说明 |
|------|------|
| 配置编辑页面 | 在 Web UI 中编辑 `policy.yaml`，支持保存到文件 |
| 启动日志页面 | 显示服务启动日志，方便排查 |
| 中文/英文切换 | 根据浏览器语言自动切换 |
| 移动端适配 | 导航栏在小屏设备上折叠 |
| API 密钥配置 | 在 Web UI 中配置和管理 API 密钥 |

## v0.3.0 — 企业特性 (8 周)

**目标**: 满足企业部署需求

### 4. 数据库持久化
| 任务 | 说明 |
|------|------|
| PostgreSQL 支持 | 添加 Spring Data JPA，替换文件 JSON 存储 |
| 策略表结构 | `roles`、`commands`、`policies` 表设计 |
| 数据迁移脚本 | 从 JSON 文件迁移到 PostgreSQL |
| Docker Compose 集成 | 添加 PostgreSQL 容器和服务编排 |

### 5. 多节点管理
| 任务 | 说明 |
|------|------|
| 节点注册 API | `POST /api/v1/admin/nodes` 注册 Sandbox 实例 |
| 节点心跳 | 每个节点定期上报健康状态 |
| 集中策略下发 | 中央控制台修改策略 → 推送到所有节点 |
| 聚合审计日志 | 所有节点的日志汇总到中央数据库 |
| 节点管理页面 | Web UI 中查看/管理所有节点 |

### 6. LDAP/AD 认证
| 任务 | 说明 |
|------|------|
| Spring Security LDAP 集成 | 添加依赖和配置 |
| LDAP 配置页面 | Web UI 中配置 LDAP 服务器地址、Base DN |
| 角色映射 | LDAP 组 → Sandbox 角色映射 |
| 自动登录 | 使用 Windows 当前用户自动登录 |

## 版本时间线

```
v0.2.0 ─── v0.3.0 ─── v0.4.0
   │           │           │
   │ 扩展稳定    │ 企业特性    │ 生态拓展
   │ 部署标准    │ 数据库      │ 多平台
   │ Web 完善    │ 多节点      │ 市场上架
   └─── 4周 ────└─── 8周 ────└─── 12周
```

当前可立即开始的优先级:
1. **VSCode 扩展稳定性** — 添加诊断日志、自动重连、角色选择 QuickPick
2. **PostgreSQL 持久化** — 替换 JSON 文件存储
3. **Windows 服务注册** — `nssm` 一键安装为系统服务
