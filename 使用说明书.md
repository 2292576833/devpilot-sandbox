## 1. 产品简介

### 1.1 什么是 DevPilot Sandbox？

DevPilot Sandbox 是一个**本地安全的执行沙箱中间件**，专门为 AI Agent 提供文件访问和命令执行的权限控制。

当 AI 编程助手（如 Cursor、Copilot、Codex）操作你的本地文件或执行命令时，DevPilot Sandbox 会在中间进行拦截和校验，确保所有操作都在安全边界内。

### 1.2 解决的核心问题

| 问题 | 说明 |
|------|------|
| ❌ AI 误删文件 | AI 可能执行 `rm -rf /` 或删除重要文件 |
| ❌ 路径穿越攻击 | AI 可能访问 `/etc/passwd` 等敏感文件 |
| ❌ 危险命令执行 | AI 可能安装恶意软件或执行危险操作 |
| ❌ 合规要求 | 企业需要审计和记录 AI 的所有操作 |

### 1.3 架构

```
AI Agent ──HTTP──▶ DevPilot Sandbox (Sidecar 服务) 
                      │ 端口 9091
                      ├── Policy Engine (YAML 策略)
                      ├── PathGuard (文件围栏) 
                      ├── CommandGuard (命令围栏)
                      └── Audit Logger (审计日志)
```

## 3. 启动服务

### 3.1 方法一：双击 run.bat（推荐）

```
双击项目目录下的 run.bat
```

脚本会自动：
1. 查找 Java 环境
2. 自动检测空闲端口（从 9091 开始）
3. 启动服务
4. 等待就绪后打开浏览器

### 3.2 方法二：命令行启动

```powershell

```

### 3.3 验证服务

```powershell
curl.exe http://127.0.0.1:9091/api/v1/guard/health
```

预期响应：
```json
{"status":"UP","version":"0.1.0","roles":["CODE_ENGINEER","DESKTOP_OPERATOR","READONLY"]}
```

### ⚠️ 重要提示

- **请使用 `127.0.0.1:9091` 而不是 `127.0.0.1:9091`**
- 端口 9090 可能被 Clash 等代理软件占用
- 使用 `127.0.0.1` 而非 `localhost` 避免浏览器 DNS 解析问题

## 5. API 参考

所有 API 端点：`http://127.0.0.1:9091/api/v1/guard/`

### 5.1 健康检查

```powershell
curl.exe http://127.0.0.1:9091/api/v1/guard/health
```

### 5.2 文件读取校验

```powershell
# 请求
curl.exe -X POST http://127.0.0.1:9091/api/v1/guard/file-read ^
  -H "Content-Type: application/json" ^
  -d "{\"roleId\":\"CODE_ENGINEER\",\"path\":\"src/Main.java\"}"

# 响应（允许）
{"allowed":true,"resolvedPath":"C:\\project\\src\\Main.java","reason":null}

# 响应（拒绝 - 路径穿越）
{"allowed":false,"resolvedPath":null,"reason":"Access denied：路径 '../../etc/passwd' is outside work directory"}
```

### 5.3 文件写入校验

```powershell
curl.exe -X POST http://127.0.0.1:9091/api/v1/guard/file-write ^
  -H "Content-Type: application/json" ^
  -d "{\"roleId\":\"CODE_ENGINEER\",\"path\":\"target/output.txt\"}"
```

### 5.4 命令执行校验

```powershell
# 合法命令
curl.exe -X POST http://127.0.0.1:9091/api/v1/guard/command-run ^
  -H "Content-Type: application/json" ^
  -d "{\"roleId\":\"CODE_ENGINEER\",\"command\":\"git status\"}"

# 被禁止的命令（带 -g 参数）
curl.exe -X POST http://127.0.0.1:9091/api/v1/guard/command-run ^
  -H "Content-Type: application/json" ^
  -d "{\"roleId\":\"CODE_ENGINEER\",\"command\":\"npm install -g express\"}"
```

### 5.5 审计日志查询

```powershell
# 查询全部
curl.exe http://127.0.0.1:9091/api/v1/guard/audit

# 按角色筛选
curl.exe "http://127.0.0.1:9091/api/v1/guard/audit?role=CODE_ENGINEER"

# 按操作筛选 + 限制条数
curl.exe "http://127.0.0.1:9091/api/v1/guard/audit?operation=file-read&limit=10"
```

### 5.6 角色管理 API

```powershell
# 列出所有角色
curl.exe http://127.0.0.1:9091/api/v1/policy/roles

# 获取单个角色
curl.exe http://127.0.0.1:9091/api/v1/policy/roles/CODE_ENGINEER

# 重载策略
curl.exe -X POST http://127.0.0.1:9091/api/v1/policy/reload
```

### 5.7 API 通用响应格式

```json
{
  "allowed": true,        // true=允许, false=拒绝
  "resolvedPath": "...",  // 解析后的绝对路径
  "reason": null          // 拒绝原因（允许时为 null）
}
```

## 6. 策略配置

### 6.1 策略文件位置

`src/main/resources/policy.yaml`

### 6.2 默认策略

```yaml
mode: multi
roles:
  - id: CODE_ENGINEER
    work_dir: "C:/Users/27736/project"
    readonly: false
    deny_files:
      - ".env"
      - "config.prod.yaml"
    global_deny_flags: ["-g", "--global"]
    allowed_paths:
      - "."
    denied_paths:
      - ".git"
    allowed_commands:
      - command: git
        subcommands: [status, add, commit, pull, push]
      - command: npm
        subcommands: [install, ci, run, test]
        denyFlags: ["-g", "--global"]
      - command: pip
        subcommands: [install, uninstall]
        denyFlags: ["--target"]
        requireEnv: ["VIRTUAL_ENV"]
    max_file_size_mb: 10
    command_timeout_sec: 120
    max_concurrency: 2

  - id: READONLY
    work_dir: "C:/Users/27736"
    readonly: true
    deny_files: [".env"]
    global_deny_flags: []
    allowed_paths:
      - "."
    allowed_commands: []
    max_file_size_mb: 5
    command_timeout_sec: 10
    max_concurrency: 1
```

### 6.3 配置字段说明

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| `id` | string | 角色唯一标识（API 调用时使用） | ✅ |
| `workDir` | string | 工作目录绝对路径 | ✅ |
| `allowedPaths` | string[] | 允许访问的路径（相对 workDir） | ❌ |
| `deniedPaths` | string[] | 禁止访问的路径 | ❌ |
| `allowedCommands` | object[] | 允许执行的命令列表 | ❌ |
| `command` | string | 命令名称 | ✅* |
| `subcommands` | string[] | 允许的子命令 | ❌ |
| `denyFlags` | string[] | 禁止的参数字段 | ❌ |
| `requireEnv` | string[] | 必须设置的环境变量 | ❌ |
| `maxFileSizeMb` | int | 最大文件大小（MB） | ❌ |
| `readonly` | boolean | 只读模式（禁止写入） | ❌ |
| `denyFiles` | string[] | 禁止访问的文件名列表 | ❌ |
| `globalDenyFlags` | string[] | 全局禁止的命令参数 | ❌ |
| `commandTimeoutSec` | int | 命令超时（秒） | ❌ |
| `maxConcurrency` | int | 最大并发数（默认1） | ❌ |
| `mode` | string | 模式: single/multi | ❌ |

> *`allowedCommands` 列表中的每个命令必须包含 `command` 字段

### 6.4 自定义策略示例

```yaml
roles:
  - id: MY_ROLE
    workDir: "D:/my-work"
    allowedPaths:
      - "."          # 允许整个工作目录
      - "data"       # 允许 data 子目录
    deniedPaths:
      - ".git"
      - "secrets"
    allowedCommands:
      - command: python
        subcommands: [script.py, main.py]
      - command: pip
        requireEnv: ["VIRTUAL_ENV"]
    maxFileSizeMb: 50
```

### 6.5 注意事项

- 修改策略后可通过 Web 控制台 `⚙️ 角色管理` 页面实时修改，或编辑 YAML 文件后重启服务
- 策略文件支持 **蛇形命名**（`work_dir`）和 **驼峰命名**（`workDir`）两种格式
- `work_dir` 必须使用**绝对路径**
- 路径分隔符使用 `/` 或 `\\`
- 角色 ID 使用大写字母和下划线
- `readonly: true` 的角色会拒绝所有写入操作
- `deny_files` 在工作区内也生效，即使路径合法
- `global_deny_flags` 对所有命令生效

## 7. 集成到 AI Agent

### 7.1 通用集成方案

DevPilot Sandbox 作为独立的 Sidecar 服务运行，任何 AI Agent 都可以通过 REST API 调用它。

### 7.2 Python 集成示例

```python
import requests

SANDBOX_URL = "http://127.0.0.1:9091/api/v1/guard"

def validate_file_access(path, mode="read"):
    """校验文件访问权限"""
    try:
        resp = requests.post(f"{SANDBOX_URL}/file-{mode}", json={
            "roleId": "CODE_ENGINEER",
            "path": path
        }, timeout=3)
        result = resp.json()
        if not result.get("allowed"):
            raise PermissionError(f"❌ 安全策略拦截: {result['reason']}")
        return result["resolvedPath"]
    except requests.ConnectionError:
        # Sandbox 未运行时，回退到正常操作
        return path

# 使用示例
try:
    safe_path = validate_file_access("src/Main.java")
    # 安全地读取文件 ...
except PermissionError as e:
    print(e)
```

### 7.3 JavaScript/TypeScript 集成示例

```typescript
const SANDBOX_URL = 'http://127.0.0.1:9091/api/v1/guard';

async function validateFileRead(path: string): Promise<string> {
  const res = await fetch(`${SANDBOX_URL}/file-read`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ roleId: 'CODE_ENGINEER', path })
  });
  const data = await res.json();
  if (!data.allowed) {
    throw new Error(`❌ 安全策略拦截: ${data.reason}`);
  }
  return data.resolvedPath;
}
```

### 7.5 集成到 Cursor/Copilot/Codex

## 9. 常见问题

### 9.1 服务无法启动

**原因**：Java 未安装或端口被占用

**解决**：
- 执行 `java -version` 确认 Java 已安装
- 执行 `netstat -ano | findstr ":9091"` 检查端口
- 手动指定端口：`java -jar target/devpilot-sandbox-0.1.0.jar --server.port=9093`

### 9.2 Web 页面显示乱码

**原因**：浏览器未按 UTF-8 解码

**解决**：确保服务启动日志中的 `Content-Type` 包含 `charset=UTF-8`，刷新浏览器

### 9.3 API 返回 404

**原因**：服务未运行或 URL 错误

**解决**：
- 访问 `http://127.0.0.1:9091/api/v1/guard/health` 确认服务运行
- 检查端口号是否正确

### 9.4 策略修改不生效

- 通过 Web 控制台 `⚙️ 角色管理` 页面实时修改，自动保存
- 修改 `policy.yaml` 后需重启服务
- `data/roles-state.json` 中的角色会覆盖 `policy.yaml` 中的同名角色

## 12. 数据持久化

通过 Web 控制台或 API 对策略做的修改（增删改角色），自动保存到 `data/roles-state.json`，服务器重启后自动恢复。

```json
[
  {
    "id": "TEST_ROLE",
    "workDir": "C:/test",
    "readonly": false,
    "denyFiles": [],
    "globalDenyFlags": [],
    "allowedCommands": [],
    "maxFileSizeMb": 10,
    "commandTimeoutSec": 30,
    "maxConcurrency": 1
  }
]
```

> **注意**：如果删除 `data/roles-state.json`，下次启动将恢复为 `policy.yaml` 中的配置。
